# 💬 카카오 로그인 인증 연동 & 안드로이드 카카오톡 웹뷰 구글 로그인 제한 우회 가이드

이 문서에서는 안드로이드 카카오톡 인앱 브라우저의 구글 로그인 차단 정책(Disallowed Useragent)에 대응하기 위해 적용한 해결 방법과, 새로 추가한 카카오 로그인(Kakao OAuth)을 실제로 Supabase와 연동하기 위해 필요한 설정 과정을 상세히 안내합니다.

---

## 🔍 문제의 원인 및 분석 (대표적인 업계 공통 이슈)

- **원인**: 구글은 보안 위협(중간자 공격 등)을 차단하기 위해 안드로이드 앱의 내부 웹뷰(커스텀 인앱 브라우저)에서 시도되는 구글 로그인을 원천 차단(`disallowed_useragent`)하고 있습니다.
- **안드로이드 카카오톡**: 카카오톡 채팅창의 링크를 클릭하여 접속하면, 정식 브라우저(크롬, 삼성인터넷)가 아닌 카카오톡 내부 웹뷰로 실행되므로 구글 로그인이 차단됩니다.
- **아이폰(iOS) 카카오톡**: iOS는 내부적으로 시스템 제공 모듈(Safari 기반)을 사용하여 구글이 정식 브라우저로 승인하므로 정상 통과됩니다.

## 🛠️ 개발 적용 사항 (우회 및 대안 제공)

이번 업데이트를 통해 두 가지 핵심 방어 코드가 적용되었습니다.

### 1. 카카오톡 외부 브라우저(아웃링크) 자동 전환 스크립트 추가
안드로이드 사용자가 카카오톡 링크로 진입하면, 웹뷰를 자동으로 닫고 스마트폰에 설정된 기본 브라우저(크롬, 삼성인터넷 등)를 열도록 강제 리다이렉트 스크립트를 삽입했습니다.
- **적용 위치**: [layout.tsx](file:///workspaces/HairGallery/app/layout.tsx) `<head>` 부분에 인라인 스크립트로 즉시 주입.
- **동작 원리**: User-Agent 분석을 통해 `kakaotalk`과 `android`가 동시에 포함된 경우, 카카오톡 전용 아웃링크 스키마(`kakaotalk://web/openExternal?url=...`)를 이용하여 화면을 전환합니다.

### 2. Supabase 기반 카카오 로그인/회원가입 기능 및 UI 추가
구글 로그인만 제공하던 기존 화면에 카카오 OAuth 기능을 추가하여, 사용자가 외부 브라우저를 타지 않더라도 카카오 간편 동의를 통해 원터치로 로그인/가입할 수 있는 대체 수단을 제공합니다.
- **적용 위치**: [page.tsx](file:///workspaces/HairGallery/app/page.tsx)
- **추가/수정 항목**:
  - `handleKakaoLogin` 및 `handleKakaoSignUp` 함수를 구현하여 Supabase OAuth(`provider: 'kakao'`) 호출.
  - 로그인 탭과 회원가입 탭에 카카오 고유의 시그니처 옐로우 디자인과 로고(`KakaoLogo`)가 적용된 버튼 배치.
  - Supabase User가 생성될 때 가입에 사용된 Provider(`google` 또는 `kakao`)를 `app_metadata`로부터 동적으로 추출하여 동기화하도록 유저 생성 로직 개선.

## 🚀 카카오 로그인 인증 연동을 위한 설정 프로세스 (3단계)

코드가 성공적으로 반영되었으므로, 실제 카카오 로그인이 작동하려면 **카카오 디벨로퍼스**와 **Supabase** 설정 페이지에서 다음 설정을 마쳐야 합니다.

### 1단계. 카카오 디벨로퍼스 설정
1. **[카카오 디벨로퍼스](https://developers.kakao.com/)**에 접속하여 로그인합니다.
2. **내 애플리케이션** -> **애플리케이션 추가하기**를 클릭하여 앱을 생성합니다.
3. 생성한 앱의 **[앱 설정] > [플랫폼 키]** 메뉴로 이동합니다. *(최근 카카오 디벨로퍼스 UI 개편으로 기존 [플랫폼] 메뉴가 [플랫폼 키]로 변경/통합되었습니다.)*
4. **JavaScript 키** 항목 옆의 **설정(또는 수정)** 아이콘을 클릭한 뒤, **[JavaScript SDK 도메인]** 부분에 서비스 URL을 등록합니다.
   - 로컬 테스트용: `http://localhost:3000`
   - 실제 배포 사이트: `https://your-domain.com` (배포된 URL)
5. **[제품 설정] > [카카오 로그인]**으로 이동하여 **활성화 설정**을 `ON`으로 바꿉니다.
6. **[제품 설정] > [카카오 로그인] > [Redirect URI]** 메뉴에서 **[Redirect URI 등록]**을 클릭하고, 아래 **Supabase의 Redirect URI**를 입력합니다:
   - URL 형식: `https://[YOUR_SUPABASE_PROJECT_REF].supabase.co/auth/v1/callback`
   - *팁: Supabase Dashboard에서 URL을 복사하여 그대로 등록하면 됩니다 (2단계 설명 참조).*
7. **[제품 설정] > [카카오 로그인] > [동의항목]**으로 이동합니다.
   - **프로필 정보(닉네임)** 및 **카카오계정(이메일)** 등의 동의 설정을 필요에 따라 설정합니다. (사용자 프로필 생성을 위해 **이메일**은 '선택 동의' 혹은 '필수 동의'로 설정하는 것을 권장합니다.)
8. **[앱 설정] > [플랫폼 키] > [REST API 키]** 항목 아래의 **[클라이언트 시크릿]** 버튼을 클릭하여 코드를 생성(발급)하고, 활성화 상태를 `ON`으로 변경합니다. *(최근 카카오 디벨로퍼스 UI 개편으로 기존 [제품 설정] > [카카오 로그인] > [보안] 메뉴의 기능이 [플랫폼 키] 내부로 이동하였습니다.)*

### 2단계. Supabase Dashboard 설정
1. **[Supabase Dashboard](https://supabase.com/dashboard)**에 로그인합니다.
2. 귀하의 프로젝트를 선택한 후, 왼쪽 사이드바에서 **[Authentication] > [Providers]**로 이동합니다.
3. 제공업체 목록 중 **Kakao**를 찾아 펼친 후 **Enable Kakao Provider**를 활성화합니다.
4. 카카오 디벨로퍼스에서 발급받은 키를 복사하여 Supabase에 붙여넣습니다:
   - **Kakao Client ID**: 카카오 디벨로퍼스 **[앱 설정] > [앱 키]** 페이지의 **REST API 키**
   - **Kakao Client Secret**: 카카오 디벨로퍼스 **[앱 설정] > [플랫폼 키] > [REST API 키] > [클라이언트 시크릿]**에서 발급받은 **Client Secret** 문자열
5. Supabase Provider 설정 화면 하단에 표시된 **Callback URL (Redirect URI)**을 복사하여, 앞선 **1단계의 6번 과정(카카오 디벨로퍼스의 Redirect URI)**에 등록해줍니다.
6. Supabase 설정 창에서 **Save**를 클릭하여 저장합니다.

### 3단계. 연동 및 테스트
- 로컬 환경(`npm run dev`) 또는 실서버 배포 후, 로그인 모달에서 **카카오 계정으로 로그인** 버튼을 눌러 카카오 동의창이 잘 출력되고 프로필 생성이 이루어지는지 검증합니다.
